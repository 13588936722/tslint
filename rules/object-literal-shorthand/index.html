--- ruleName: object-literal-shorthand description: Enforces/disallows use of ES6 object literal shorthand. hasFix: true optionsDescription: |- `"always"` assumed to be default option, thus with no options provided the rule enforces object literal methods and properties shorthands. With `"never"` option provided, any shorthand object literal syntax causes an error. The rule can be configured in a more granular way.
